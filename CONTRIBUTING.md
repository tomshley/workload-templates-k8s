# Contributing

Thank you for your interest in contributing to **Tomshley Workload Templates for Kubernetes**.

This project is part of **Tomshley LLC's OSS and IP strategy** and is intentionally opinionated.

## Philosophy

- Composable, reusable Kubernetes manifests
- Clear separation between workloads, components, and examples
- Convention over configuration
- Production-ready defaults with explicit extension points

Changes that increase complexity, size, or scope without strong justification are unlikely to be accepted.

## What Contributions Are Welcome

- Bug fixes
- Security improvements
- Documentation improvements
- New workload or component templates that clearly fit the layering model

## What Is Out of Scope

- Application-specific configuration or secrets
- Cloud-provider-specific resources (use separate modules)
- Templates that duplicate existing workload patterns
- CI logic changes unrelated to template validation

## Process

1. Open an issue describing the change
2. Fork the repository
3. Create a focused merge request
4. Keep commits clean and scoped

Tomshley LLC reserves final decision-making authority on all contributions.
